## âš ï¸ Cosmetic Issues (Low Priority)

### 1. "Unknown fields" warning for `_meta` block

```
Unknown fields in openai-fast-check.yml: _meta
```

- **Cause**: Installed evaluator files include an `_meta` block added by `library install`
- **Impact**: Cosmetic only â€” evaluators still run correctly
- **Suggestion**: Either suppress warning for `_meta`, or parse/ignore it

**Installed file structure** (what `library install` creates):
```yaml
_meta:
  source: adversarial-evaluator-library
  source_path: openai/fast-check
  version: "1.2.0"
  installed: "2026-02-06T11:37:09Z"

name: fast-check
model: gpt-4o-mini
# ... rest of evaluator
```

### 2. "Unknown fields" may also apply to `model_requirement`

Our evaluators include both legacy and Phase 2 fields:
```yaml
model: gpt-4o-mini              # Legacy (used by v0.9.0)
api_key_env: OPENAI_API_KEY     # Legacy (used by v0.9.0)
model_requirement:              # Phase 2 (not yet used)
  family: gpt
  tier: mini
  min_version: "4o-mini"
```

The warning appears once per evaluator load. Consider:
- Suppress for known future fields (`model_requirement`, `_meta`)
- Or add `--quiet` flag to suppress all warnings
